# Модель данных

## 1. Общая информация

База данных: MySQL 8.0  
Назначение: хранение данных пользователей и игрового процесса  

Модель данных разработана для поддержки:

- пользователей и ролей;
- прогресса игрока;
- игровых сессий (вылазок);
- результатов вылазок;
- игровых событий;
- инвентаря игрока.

---

## 2. Основные сущности

В системе используются следующие основные таблицы:

- users  
- roles  
- user_roles  
- player_progress  
- inventory  
- runs  
- run_results  
- events  

---

## 3. Описание таблиц

### 3.1 Users

Таблица пользователей системы.

Поля:

- id — уникальный идентификатор  
- email — email пользователя  
- password_hash — хэш пароля  
- nickname — имя игрока  
- created_at — дата регистрации  
- last_login_at — последний вход  
- is_banned — статус блокировки  

---

### 3.2 Roles

Таблица ролей пользователей.

Поля:

- id  
- name  

Примеры ролей:

- player  
- admin  

---

### 3.3 User Roles

Связующая таблица между пользователями и ролями.

Поля:

- user_id  
- role_id  
- assigned_at  

Позволяет одному пользователю иметь несколько ролей.

---

### 3.4 Player Progress

Таблица прогресса игрока.

Поля:

- user_id  
- level — уровень  
- xp — опыт  
- soft_currency — внутриигровая валюта  
- hard_currency — премиум валюта  
- updated_at  

Данные обновляются после завершения вылазок.

---

### 3.5 Inventory

Таблица инвентаря игрока.

Поля:

- id  
- user_id  
- item_code — код предмета  
- quantity — количество  
- updated_at  

Используется для хранения ресурсов и предметов.

---

### 3.6 Runs

Таблица игровых сессий (вылазок).

Поля:

- id  
- user_id  
- status — состояние  
- started_at — время начала  
- ended_at — время окончания  

Возможные статусы:

- started  
- extracted  
- dead  

---

### 3.7 Run Results

Таблица результатов вылазок.

Поля:

- run_id  
- is_success — успешность  
- rooms_visited — посещённые комнаты  
- loot_value — ценность добычи  
- duration_seconds — длительность  
- xp_gained — полученный опыт  
- soft_currency_gained — полученная валюта  
- ended_reason — причина завершения  

Возможные значения ended_reason:

- exit  
- death  

---

### 3.8 Events

Таблица игровых событий.

Поля:

- id  
- user_id  
- run_id  
- event_type — тип события  
- payload_json — дополнительные данные  
- created_at — время  

Примеры событий:

- room_enter  
- loot_found  
- enemy_killed  

---

## 4. Связи между сущностями

Основные связи:

- пользователь → прогресс (1 к 1)  
- пользователь → вылазки (1 ко многим)  
- вылазка → результат (1 к 1)  
- вылазка → события (1 ко многим)  
- пользователь → события (1 ко многим)  
- пользователь → инвентарь (1 ко многим)  

---

## 5. Логика хранения данных

Система построена таким образом, что:

- все ключевые действия игрока сохраняются;
- прогресс не теряется между сессиями;
- каждая вылазка фиксируется отдельно;
- события позволяют анализировать поведение игрока.

---

## 6. Практическое использование

Модель данных используется для:

- авторизации пользователей;
- получения профиля;
- запуска вылазки;
- завершения вылазки;
- начисления наград;
- логирования событий.

---

## 7. Вывод

Модель данных охватывает все основные аспекты игрового процесса и обеспечивает корректное хранение информации о пользователях, их действиях и прогрессе.
